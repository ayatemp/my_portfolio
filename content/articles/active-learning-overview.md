## はじめに

機械学習モデルの性能はデータの量と質に依存する。しかし、高品質なラベル付きデータを大量に準備することは、コスト・時間・専門知識の面で多くの現場で困難を伴う。

**Active Learning（能動学習）** は、この問題に対するエレガントな解答のひとつだ。モデルが「どのデータにラベルを付けてほしいか」を自ら問い合わせることで、少ないアノテーション予算で最大限の精度向上を実現する。

---

## Active Learningの基本構造

Active Learningのループは以下の3ステップで回る：

1. **学習**: 現在のラベル付きデータでモデルを訓練
2. **クエリ**: 未ラベルプールから「最も情報量の多い」サンプルを選択
3. **アノテーション**: 人間のアノテータがラベルを付与

このサイクルを繰り返すことで、ランダムサンプリングと比べて**同じ精度を達成するのに必要なラベル数を大幅に削減**できる。

---

## 主要なクエリ戦略

### 1. Uncertainty Sampling

最もシンプルかつ広く使われる手法。モデルが最も自信を持てないサンプルを選ぶ。

```python
def least_confident_query(proba: np.ndarray) -> np.ndarray:
    """最も予測確信度が低いサンプルのインデックスを返す"""
    max_proba = proba.max(axis=1)
    return np.argsort(max_proba)  # 昇順 = 確信度が低い順
```

代表的な指標：
- **Least Confident**: 最大クラス確率が最も低いもの
- **Margin Sampling**: 上位2クラスの確率差が最も小さいもの
- **Entropy**: 予測分布のエントロピーが最大のもの

### 2. Query by Committee (QBC)

複数のモデル（委員会）を訓練し、予測が最も一致しないサンプルを選ぶ。
**モデル間の不一致 = 学習が最も必要な領域**という直感に基づく。

### 3. Core-Set Approach

未ラベルデータの特徴空間をカバーするように幾何学的に選択する。
ラベルなしデータに対して**貪欲なk-センター問題**として定式化される。

## 実務での注意点

### Distribution Shift の罠

Active Learningで選ばれるサンプルは、確信度が低い＝境界近傍に偏る。
これは**モデルの学習データが分布的に偏る**ことを意味し、予期せぬ性能劣化を招くことがある。

定期的にランダムサンプルを混ぜる、Core-Setとハイブリッドにするなどの対策が有効だ。

### アノテータの疲弊

難しいサンプルを集中的に提示されるアノテータは、品質が低下しやすい。
クエリ戦略をアノテータの疲弊と組み合わせてモデリングする研究も進んでいる。

## まとめ

Active Learningは、単なるサンプリング手法ではなく**データ効率の思想**だ。
どのクエリ戦略を選ぶかはタスクとデータに依存するが、まず `entropy sampling` + `random mixing` の組み合わせから試すことを推奨する。

次回は、Uncertainty Samplingをディープラーニングに適用するための **Bayesian Deep Learning** との接続について書く予定だ。
